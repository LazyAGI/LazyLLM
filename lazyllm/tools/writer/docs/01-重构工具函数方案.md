# 重构工具函数方案：迁移至 Writer IR

## 1. 背景与目标

`lazyllm/tools/writer/data_models/writer_ir.py` 定义了新的中间表示（Writer IR），用一套
`WriterDocument` / `WriterBlock` / `WriterAuthoring` / `WriterConstraints` / `WriterSpan`
统一掉历史的多套并行模型。

本次重构把 `lazyllm/tools/writer/tools/` 下所有工具函数的入参与产物，从旧模型迁移到 Writer IR。

### 重构原则

- 破坏性改动，不做旧版本兼容适配。
- 旧类型（`WritingOutline` / `DraftDocument` / `WritingOutput` / `DocIR` 等）在迁移完成后后续会被删除，本次不保留别名或兼容层。
- 入参类型在 `_unified_model` 一侧直接指向新的 Writer IR 类型。

### 术语

`node_id` 是 Writer IR 的内部稳定标识；本次统一用它替换历史的 `block_id` / `section_id` / `outline_id` 等定位字段。`provider_binding.*` 才放外部系统 ID。

## 2. 类型替换映射（旧 → 新）

| 旧类型 | 新类型 | 说明 |
| --- | --- | --- |
| `OutlineNodeConstraints` | `WriterConstraints` | 字段一一对应；旧的 `source_refs` 下沉到 `WriterBlock.source_refs` |
| `OutlineNode` | `WriterBlock`（stage=`outline`） | `title`→`content`，`level`→`numbering.level`，`instruction`→`authoring.instruction` |
| `WritingOutline` | `WriterDocument`（stage=`outline`） | `outline_id`→`document_id`，`nodes`→`blocks` |
| `SectionInstruction` | `WriterAuthoring` + `WriterConstraints` | 见下方字段表 |
| `SectionInstructionList` | 取消 | 指令直接挂在 `WriterBlock.authoring` 上，不再有独立列表 |
| `DraftBlock` | `WriterBlock`（stage=`draft`） | `block_id`→`node_id`，`content`→`content` |
| `DraftSection` | `WriterBlock`（stage=`draft`） | `sub_sections`/`blocks` 合并进 `children` |
| `DraftDocument` | `WriterDocument`（stage=`draft`） | `draft_id`→`document_id`，`sections`→`blocks` |
| `WritingOutput` | `WriterDocument`（stage=`final`） | `content` 由 blocks 渲染产出，`output_format` 进 `metadata` |
| `DocBlock` | `WriterBlock` | `block_id`→`node_id`，`block_type`→`type`，`text`→`content` |
| `DocSpan` | `WriterSpan` | `stype`→`style` |
| `DocIR` | `WriterDocument` | `doc_id`→`document_id`，`adapter`→`provider_binding.adapter` |

### `SectionInstruction` 字段映射

| `SectionInstruction` 字段 | 目标位置 |
| --- | --- |
| `instruction_id` | `WriterAuthoring.instruction_id` |
| `outline_node_id` | `WriterAuthoring.origin_node_id`（同时作为宿主 `WriterBlock.node_id` 的来源） |
| `section_title` | 宿主 `WriterBlock.content` |
| `section_goal` | `WriterAuthoring.instruction` 或 `WriterConstraints.section_goal` |
| `required_points` | `WriterConstraints.required_points` |
| `source_refs` | `WriterBlock.source_refs` |
| `fact_constraints` | `WriterConstraints.fact_constraints` |
| `style_constraint` | `WriterConstraints.style_constraints` |
| `relation_constraints` | `WriterConstraints.relation_constraints` |
| `visual_needs` | `WriterAuthoring.visual_needs` |
| `expected_blocks` | `WriterAuthoring.expected_blocks` |
| `pending_subtasks` | `WriterAuthoring.pending_subtasks` |
| `revision_notes` | `WriterAuthoring.revision_notes` |
| `meta` | `WriterAuthoring.meta` |

### 关于 `WritingSubTask`

`WriterAuthoring.pending_subtasks` 是 `List[str]`，与旧 `writing.py` 中 `DraftBlock.subtasks: List[WritingSubTask]`
的字段类型并不一致。现有 `writing.py` 自身的 `pending_subtasks` 已是 `List[str]`，且 `WritingSubTask`
在当前代码中尚未大量投入使用。本次按现有逻辑扁平化：subtask 的描述字符串进 `pending_subtasks`，
更丰富的 `subtask_type` / `placeholder` / `status` 暂不保留。若后续需要恢复子任务状态，再单独讨论扩展。

## 4. 函数级 / 参数级改动明细

每一项按 `文件 → 函数 → 参数/产物` 三层列出。`→` 左侧为旧，右侧为新。

### 4.1 `tools/planning_tools.py`

#### `generate_outline`

- 入参
  - `task: Any` → 不变（`WritingTask`）
  - `context: Any` → 不变（`WritingContext`）
  - `resource_profiles: Any` → 不变（`ResourceProfile` 列表）
  - `execution_results: Any` → 不变
- LLM 结构化产出：`WritingOutline` → `WriterDocument`（stage=`outline`）
- 工具产物 artifact：`outline`（`WritingOutline`）→ `outline`（`WriterDocument`）
- 内部归一化 `_normalize_outline*`：`OutlineNode` 树 → `WriterBlock` 树，约束写入 `authoring.constraints`

#### `generate_section_instructions`

- 入参
  - `outline: Any`（`WritingOutline`）→ `WriterDocument`（stage=`outline`）
  - `context: Any` → 不变
  - `execution_results: Any` → 不变
- LLM 结构化产出：`SectionInstructionList` → 直接产出挂在 `WriterBlock.authoring` 上的指令（即 stage=`outline` 的 `WriterDocument`，每个 block 的 `authoring` 被填充）
- 工具产物 artifact：`section_instructions`（`SectionInstructionList`）→ `outline_with_instructions`（`WriterDocument`，`authoring` 已填充）
- 删除私有方法 `_instruction_target_nodes` / `_normalize_section_instruction*` 中对 `SectionInstruction` 的依赖，改为直接操作 `WriterBlock.authoring`

### 4.2 `tools/drafting_tools.py`

#### `generate_draft_section`

- 入参
  - `task: Any` → 不变
  - `section_instruction: Any`（`SectionInstruction` / `SectionInstructionList`）→ `outline_block: Any`（`WriterBlock`，stage=`outline`，已带 `authoring`）
  - `context: Any` → 不变
  - `previous_sections: Any` → `previous_blocks: Any`（`WriterBlock` 列表或 `WriterDocument`）
- LLM 结构化产出：`DraftSection` → `WriterBlock`（stage=`draft`，`children` 为内容块）
- 工具产物 artifact：`draft_section` → `draft_block`（`WriterBlock`）
- `_unified_section_instruction` → `_unified_outline_block`

#### `generate_draft_document`

- 入参
  - `draft_sections: Any`（`DraftSection` 列表）→ `draft_blocks: Any`（`WriterBlock` 列表，或 stage=`draft` 的 `WriterDocument`）
  - `context: Any` → 不变
  - `outline: Any`（`WritingOutline`）→ `outline: Any`（`WriterDocument`，stage=`outline`）
  - `title: Any` → 不变
- 工具产物 artifact：`draft_document`（`DraftDocument`）→ `draft_document`（`WriterDocument`，stage=`draft`）

#### `generate_writing_output`

- 入参
  - `draft: Any`（`DraftDocument`）→ `draft: Any`（`WriterDocument`，stage=`draft`）
  - `context: Any` → 不变
  - `output_format: str` → 不变（仅 `markdown`）
- 工具产物 artifact：`writing_output`（`WritingOutput`）→ `final_document`（`WriterDocument`，stage=`final`）
- markdown 文本仍由 block 树渲染产出，写入 `metadata.output_format`，并继续落 `.md` 文件

### 4.3 `tools/revision_tools.py`

> revision 模型层（`data_models/revision.py`）已有 `target_node_id` / `target_block_id` 双字段迁移脚手架，本次拆掉 `target_block_id` 与同步 validator，统一为 `target_node_id`。

#### `locate_revision_target`

- 入参
  - `task: Any` → 不变
  - `doc_ir: Any`（`DocIR`）→ `document: Any`（`WriterDocument`）
  - `context: Any` → 不变
- 候选 id：`block_id` 集合 → `node_id` 集合（`WriterDocument.iter_blocks()`）
- LLM 结构化产出 `LocateResult`：`target_block_ids` 字段删除，仅保留 `target_node_ids`

#### `generate_modify_plan`

- 入参
  - `task: Any` → 不变
  - `doc_ir: Any`（`DocIR`）→ `document: Any`（`WriterDocument`）
  - `locate_result: Any` → 不变（`LocateResult`，已用 `target_node_ids`）
  - `context: Any` → 不变
- 内部 `focused_doc`（`DocIR`）→ 聚焦后的 `WriterDocument`
- `ModifyInstruction.target_block_id` 字段删除

#### `generate_patch_set`

- 入参
  - `doc_ir: Any`（`DocIR`）→ `document: Any`（`WriterDocument`）
  - `modify_plan: Any` → 不变（已用 `target_node_id`）
  - `context: Any` → 不变
- `PatchHunk.target_block_id` 字段删除；`Anchor.block_id` → `Anchor.node_id`
- hunk 构造中 `block.text` → `block.content`

#### `apply_patch`

- 入参
  - `doc_ir: Any`（`DocIR`）→ `document: Any`（`WriterDocument`）
  - `patch_set: Any` → 不变
  - `context: Any` → 不变
- 产物 artifact：`revised_doc_ir`（`DocIR`）→ `revised_document`（`WriterDocument`）
- 应用逻辑里 `block_id` 查找 → `node_id` 查找；insert/replace/delete 目标改为 `WriterBlock`

#### 删除：`draft_to_doc_ir`

- 统一为单一 IR 后，draft↔doc_ir 互转不再需要，整体删除。

#### 删除：`doc_ir_to_draft`

- 同上，整体删除。

### 4.4 `tools/context_tools.py`

#### `create_writing_context`

- 入参
  - `task: Any` → 不变
  - `resource_profiles: Any` → 不变
  - `doc_ir: Any`（`DocIR`）→ `document: Any`（`WriterDocument`，任意 stage）
- `block_summaries` 构造：遍历 `DocBlock` → 遍历 `WriterBlock`，`block_id` → `node_id`
- `document_summary.structure_summary`：heading 块的 `block_type=='heading'` 判断 → `type=='heading'`

#### `update_writing_context`

- 入参
  - `artifacts: Any` → 不变（多态），但其中 `DraftSection` / `DraftDocument` / `WritingOutline` 实例 → 对应 stage 的 `WriterDocument` / `WriterBlock`
  - `context: Any` → 不变（`WritingContext`，但其 `outline` / `draft_sections` / `draft_document` 字段类型已改为 Writer IR）
- `_resolve_artifact_kind` 的 schema 名匹配需同步更新

### 4.5 `tools/quality_tools.py`

#### `validate_section`

- 入参
  - `draft_section: Any`（`DraftSection`）→ `draft_block: Any`（`WriterBlock`，stage=`draft`）
  - `section_instruction: Any`（`SectionInstructionList`）→ `outline_document: Any`（`WriterDocument`，stage=`outline`，`authoring` 已填充）
  - `context: Any` → 不变
- `_match_instruction`：从 `SectionInstructionList` 匹配 → 从 outline `WriterDocument` 的 block `authoring` 匹配

#### `validate_draft_document`

- 入参
  - `draft_document: Any`（`DraftDocument`）→ `draft_document: Any`（`WriterDocument`，stage=`draft`）
  - `context: Any` → 不变

#### `validate_patch_set`

- 入参不变（`patch_set` / `context` / `task`），但 hunk 序列化随 `PatchHunk` 字段变化（`target_block_id` 移除）

### 4.6 `tools/resource_tools.py`

#### `document_to_docir`

- 入参
  - `target_document: Any` → 不变（`TargetDocument`）
  - `context: Any` → 不变
- 工具产物 artifact：`doc_ir`（`DocIR`）→ `document`（`WriterDocument`，stage 视来源而定，通常 `final`）
- 内部 `DocBlock` 构造 → `WriterBlock` 构造；`block_type` → `type`，`text` → `content`，`block_id` → `node_id`
- 飞书等 adapter 的 `_BLOCK_TYPE_MAPS` 输出值保持，但写入 `WriterBlock.type`

#### `write_to_document`

- 入参
  - `content: Any`（`WritingOutput`）→ `content: Any`（`WriterDocument`，stage=`final`）
  - `target_document: Any` → 不变
- 取正文：`output.content` → 从 `WriterDocument` 渲染 markdown

#### `profile_resources`（不在本次范围）

- 不涉及上述四类模型，不改动。

## 5. revision 模型字段收敛明细

`data_models/revision.py` 当前为迁移期双字段结构，本次收敛为单字段：

| 模型 | 删除字段 | 保留字段 |
| --- | --- | --- |
| `LocateResult` | `target_block_ids` | `target_node_ids` |
| `ModifyInstruction` | `target_block_id` | `target_node_id` |
| `ModifyPlan` | `target_block_ids` | `target_node_ids` |
| `PatchHunk` | `target_block_id` | `target_node_id` |
| `Anchor` | `block_id` | `node_id` |

同步删除各模型上的 `normalize_target_id*` validator（双字段同步逻辑不再需要）。

删除 `SECTION_META_FIELDS` / `BLOCK_META_FIELDS`（仅被 `doc_ir_to_draft` / `draft_to_doc_ir` 使用，随转换函数一起删除）。

## 6. 不在本次范围

- `WritingTask` / `InputResource` / `TargetDocument` / `Selection`（task.py）不动。
- `ResourceProfile` / `MaterialStyle`（resource.py）不动。
- `WritingContext` 自身的非文档字段（`facts` / `style_profile` / `block_summaries` 等）类型不动，仅 `outline` / `draft_sections` / `draft_document` 三个文档槽位换类型。`BlockSummary.block_id` 随术语统一改 `node_id`。
- `quality.py` 的 `AuditResult` / `ReviewReport` / `AuditIssue` 不动。
- `multimodal.py` 不动。
- 旧类型文件（`writing.py` / `docir.py`）的删除作为迁移完成的收尾步骤，待工具层全部切换并验证后再删，避免中途 import 断裂。
